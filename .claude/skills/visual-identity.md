# Visual Identity — Anti-Generic Design Standards

**Layer 2 skill.** Applies when visual design matters. Must respect Layer 1 (Engineering Discipline) constraints at all times.

## Purpose

Prevent generic, template-grade output. Every interface Claude produces must have intentional visual identity — distinct typography, deliberate color, considered layout. The goal is work that looks designed by a human with opinions, not generated by a machine with defaults.

---

## Activation

This skill activates whenever Claude generates visible UI: landing pages, dashboards, portfolios, game interfaces, marketing sites, component libraries, or any project where visual presentation matters. It does NOT activate for purely structural work (APIs, CLIs, build tooling).

---

## The Anti-Generic Checklist

Before delivering any visual output, verify all of the following:

- [ ] No default system fonts (Inter, Roboto, Arial) as the primary typeface
- [ ] No purple-on-white gradient as the dominant color scheme
- [ ] No uniform `border-radius` on every element
- [ ] No center-aligned-everything layout
- [ ] No generic hero + three-column features template
- [ ] No identical box shadows on every card
- [ ] No stock illustration aesthetic
- [ ] Font pairing differs from the last project delivered
- [ ] Color palette differs from the last project delivered

If any item fails, redesign before delivering.

## Typography Standards

### Selection Rules

1. **Choose distinctive typefaces.** Every project gets a deliberate display + body font pairing. Never fall back to system fonts or ubiquitous defaults without explicit justification.
2. **Pair with contrast.** Display font provides character. Body font provides readability. They should complement, not match.
3. **Vary across projects.** Do not reuse the same pairing across different deliverables.

### Typography Scale

Use fluid type with `clamp()`. Define scale as CSS custom properties:

```css
:root {
  --text-xs: clamp(0.75rem, 0.7rem + 0.25vw, 0.875rem);
  --text-sm: clamp(0.875rem, 0.8rem + 0.375vw, 1rem);
  --text-base: clamp(1rem, 0.9rem + 0.5vw, 1.125rem);
  --text-lg: clamp(1.125rem, 1rem + 0.625vw, 1.25rem);
  --text-xl: clamp(1.25rem, 1rem + 1.25vw, 1.5rem);
  --text-2xl: clamp(1.5rem, 1rem + 2.5vw, 2rem);
  --text-3xl: clamp(1.875rem, 1rem + 4.375vw, 3rem);
  --text-4xl: clamp(2.25rem, 1rem + 6.25vw, 4rem);
  --text-hero: clamp(3rem, 2rem + 10vw, 12rem);
}
```

### Typography CSS Standards

```css
/* Headlines: tight, balanced */
.headline {
  letter-spacing: -0.02em;
  line-height: 1;
  text-wrap: balance;
}

/* Body: comfortable reading */
.body {
  letter-spacing: 0.01em;
  line-height: 1.6;
  max-width: 65ch;
}

/* Overlines: tracked uppercase */
.overline {
  text-transform: uppercase;
  letter-spacing: 0.15em;
  font-size: var(--text-xs);
  font-weight: 500;
}

/* Refined rendering */
.refined-text {
  font-feature-settings: "kern" 1, "liga" 1, "calt" 1;
  text-rendering: optimizeLegibility;
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}
```

### Font Loading

Always preload critical fonts. Always use `font-display: swap`. Always define fallback stacks:

```html
<link rel="preload" href="/fonts/Display-Bold.woff2" as="font" type="font/woff2" crossorigin>
```

```css
:root {
  --font-display: 'Clash Display', 'Arial Black', sans-serif;
  --font-body: 'Plus Jakarta Sans', 'Helvetica Neue', sans-serif;
  --font-mono: 'JetBrains Mono', 'Consolas', monospace;
}
```

## Color System

### Rules

1. **Define a complete palette as CSS custom properties.** Never hardcode color values.
2. **Establish hierarchy.** One dominant color, one or two accents, neutral backgrounds. Do not distribute colors evenly — create visual weight.
3. **Support dark mode.** Use `prefers-color-scheme` or `color-scheme: light dark`. Define separate token sets for each mode.
4. **Avoid the default AI palette.** No purple-to-blue gradients on white backgrounds unless the project specifically calls for it.

### Color Techniques

- Use `color-mix()` for tints and shades instead of opacity hacks.
- Use `oklch()` or `hsl()` for perceptually uniform color relationships.
- Use mesh gradients (multiple `radial-gradient` layers) for organic, non-linear color fields.
- Use animated gradients sparingly and only with `prefers-reduced-motion` respect.

## Layout

### Rules

1. **Break symmetry intentionally.** Not every section needs centered content. Use asymmetric compositions, offset grids, and deliberate whitespace imbalance.
2. **Use CSS Grid for structure.** Define explicit grid systems with named areas. Vary column structures across sections.
3. **Create depth.** Overlapping elements, layered shadows, z-index stacking, and transparency create visual hierarchy beyond flat layouts.
4. **Generous whitespace or intentional density.** Choose one. Never land in the mushy middle.

### Techniques

- Overlapping elements via negative margins or grid overlap
- Diagonal/angular sections via `clip-path`
- Full-bleed media that escapes containers
- Mixed grid column counts across sections

---

## Background and Texture

Never use a plain solid-color background as the default without consideration. Options:

- **Noise/grain overlays** (SVG filter, low opacity, `pointer-events: none`)
- **Mesh gradients** (multi-point radial gradients)
- **Subtle patterns** (geometric, dot grids, line textures)
- **Glassmorphism** (`backdrop-filter: blur()` with transparency)
- **Layered shadows** (multiple `box-shadow` values at increasing offsets)

All background treatments must remain subtle. They support content — they do not compete with it.

## Design Direction

Before generating any visual interface, Claude must select a deliberate design direction. Choose one:

| Direction | Characteristics |
|---|---|
| Brutalist | Raw, exposed structure, monospace, high contrast, no decoration |
| Minimal | Extreme reduction, generous space, single accent, quiet typography |
| Retro-Futurist | CRT aesthetics, scanlines, monospace, neon accents, dark backgrounds |
| Organic/Natural | Soft curves, earth tones, flowing layouts, natural textures |
| Luxury | High contrast, serif display, restrained palette, generous whitespace |
| Playful | Bold color, rounded forms, animation, unexpected interactions |
| Editorial | Strong typographic hierarchy, column layouts, pull quotes, reading rhythm |
| Art Deco | Geometric patterns, gold accents, symmetrical ornamentation |
| Industrial | Exposed grid, utility aesthetic, tight spacing, functional color |
| Japanese Minimal | Asymmetric balance, muted palette, texture focus, quiet interactions |

Never produce output without a conscious direction. If the user does not specify one, select the most appropriate direction for the project context and state the choice.

---

## Constraint

This entire skill operates within Layer 1 boundaries:

- All typography choices must maintain WCAG contrast ratios.
- All layout choices must use semantic HTML structure.
- All color systems must support `prefers-color-scheme`.
- All font loading must not block rendering.
- All texture and background treatments must not impact performance.

Visual identity enhances the engineering foundation. It never compromises it.
